# 核心架构

## 总览

Deep Research 采用 Spring Boot 后端 + React 前端 + 多 Agent Pipeline 的分层架构。

```
前端 (React/Vite) ──HTTP──▶ REST Controller ──▶ @QueuedAsync ──▶ AgentPipeline
     ◀──SSE── SseHub ◀── EventPublisher ◀── Agent 各阶段
```

## 智能体工作流

三阶段流水线，每个阶段只负责一个清晰目标，通过 `DeepResearchState` 传递上下文：

```
用户问题
  → ScopeAgent：判断是否需要澄清，生成 ResearchBrief
  → SupervisorAgent：拆解研究计划，反复调用 conductResearch
      → ResearcherAgent：执行子主题研究，调用 tavilySearch
          → SearchAgent：Tavily 检索 + 网页摘要压缩
      → 返回研究笔记给 Supervisor
  → ReportAgent：基于全部研究笔记生成 Markdown 报告
```

## 运行时 SPI

Agent 层不直接绑定 LLM 框架，依赖 `AgentRuntime` 接口：

```
Agent（业务逻辑）
  └── AgentRuntime SPI
        ├── AgentscopeJavaAgentRuntime（默认）
        │     └── ReActAgent + Toolkit + Middleware
        └── Langchain4jAgentRuntime（备份）
              └── chatModel.chat() + 项目自建循环
```

通过 `RESEARCH_AGENT_FRAMEWORK` 环境变量切换。

## 关键类职责

| 类 | 包 | 职责 |
|---|---|---|
| `AgentPipeline` | workflow | 编排 Scope → Supervisor → Report 三阶段 |
| `DeepResearchState` | state | 集中状态：范围、预算计数、token、笔记 |
| `AgentRuntime` | runtime | LLM 框架抽象接口 |
| `ResearchChatClient` | runtime | 单次 chat + agent 循环接口 |
| `AgentscopeJavaChatClient` | runtime.agentscope | 创建 ReActAgent，驱动 streamEvents() |
| `AgentscopeStageAgentFactory` | runtime.agentscope | 构建 ReActAgent 实例 |
| `AgentscopeToolkitAdapter` | runtime.agentscope | 项目工具 → AgentScope Toolkit 适配 |
| `FixedOtelTracingMiddleware` | runtime.agentscope | 自定义 OTel 中间件，修复 Reactor 异步上下文传播 |
| `AgentscopeTraceContextMiddleware` | runtime.agentscope | 向 OTel span 注入研究元数据 |
| `ToolRegistry` | tool | 扫描注解，按阶段分组注册工具 |
| `ResearchObservation` | infra.observability | 创建 workflow/stage 级 OTel span |
| `ObservabilityConfiguration` | infra.observability | 初始化 OpenTelemetry SDK + OTLP 导出器 |
| `SseHub` | infra.sse | SSE 推送 + Redis ZSet 断线重放 |
| `ResearchTaskExecutor` | infra.async | 有界线程池，提交研究任务 |

## 分层架构

```
interfaces/        接口层：REST Controller、Service
  └── controller/  HTTP 端点
  └── service/     业务编排
application/       应用层：Agent、工具、状态、工作流
  └── agent/       5 个 Agent 实现 + runtime 适配器
  └── tool/        工具注册中心 + 注解
  └── state/       DeepResearchState
  └── workflow/    AgentPipeline
domain/            领域层：实体、MyBatis Mapper
infra/             基础设施层：异步、SSE、观测、配置
```

## 数据流

1. 前端 POST 消息 → Controller → `@QueuedAsync` 提交到线程池
2. `AgentPipeline.run()` 在线程池中执行
3. 每个阶段通过 `ResearchObservation.observeStage()` 包裹，创建 OTel span
4. Agent 内部通过 `ResearchChatClient.runAgent()` 创建 `ReActAgent`，驱动工具循环
5. 工具执行通过 `ToolRegistry` 查找，`AgentscopeToolkitAdapter` 适配为 AgentScope Toolkit
6. 事件通过 `EventPublisher` → `SseHub` → SSE 推送到前端
7. 状态变更通过 `DeepResearchState` 传递，完成时持久化到 MySQL
