# 2026-06-13 研究性能优化

## 背景

完整 HIGH 预算研究在实际验证中耗时约 43 分钟。事件线显示主要瓶颈来自两点：

- Supervisor 虽然配置了 `max-concurrent-units=2`，但模型实际串行委派 Researcher，导致研究主题不能并发执行。
- Tavily 搜索本身通常只需数秒，网页摘要阶段存在 300 秒级 LLM 超时长尾。

## 改动

1. SupervisorAgent 改为“模型规划 + 代码调度”：
   - 先让模型输出结构化 JSON 子任务列表。
   - Java 层按 `max-concurrent-units` 并发执行 ResearcherAgent。
   - 手工补齐 `execute_tool conductResearch` span，保留 Langfuse 层级。

2. SearchAgent 增加搜索处理优化：
   - URL + 内容 hash 级网页摘要缓存。
   - 摘要输入长度限制。
   - 网页摘要短超时，超时后降级为 Tavily snippet 或截断内容。
   - 单次搜索结果数硬上限。

3. AgentPipeline 状态持久化优化：
   - 进入 Supervisor 前立即写入 `IN_RESEARCH`。
   - 进入 Report 前立即写入 `IN_REPORT`。

4. 默认预算调整：
   - HIGH 保持最多 4 个研究主题。
   - HIGH 每个 Researcher 的最大搜索次数从 3 降为 2。
   - 单次 Tavily 搜索默认最多保留 3 条结果。

## 新增配置

```properties
RESEARCH_SEARCH_MAX_RESULTS_PER_QUERY=3
RESEARCH_SEARCH_SUMMARY_TIMEOUT_SECONDS=60
RESEARCH_SEARCH_SUMMARY_RAW_CONTENT_MAX_CHARS=12000
RESEARCH_SEARCH_SUMMARY_FALLBACK_CONTENT_MAX_CHARS=1200
RESEARCH_SEARCH_SUMMARY_CACHE_ENABLED=true
RESEARCH_SEARCH_SUMMARY_CACHE_TTL_MINUTES=60
RESEARCH_SEARCH_SUMMARY_CACHE_MAX_ENTRIES=1024
```
