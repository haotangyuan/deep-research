# Agent Flow 图谱增量更新与交互修复

## 背景

本次修复集中在研究详情页右侧 Agent Flow 图谱的实时交互问题。

主要表现：

- 用户在对话框发送消息后，Agent 输出期间图谱会短暂消失，输出完成后才恢复。
- 实时新增子节点时，画布会自动自适应并移动，打断用户当前观察位置。
- 点击 `ResearchAgent` 右侧加号时，第一次可能展开失败，需要再点另一个加号才生效。
- 展开后的子节点可能和已有节点重叠。
- 刷新页面后，右侧图谱可能初始落在空白区域，需要点击节点或双击画布后才恢复到可见位置。

## 根因

### 全量重绘导致图谱闪烁

原有数据刷新路径更接近“整图重新生成”。在 SSE 持续追加消息和事件时，前端频繁触发图谱重绘，G6 会重新计算画布内容与视口状态，导致图谱在 Agent 输出阶段短暂不可见。

### `render()` / `layout()` 会移动视口或已有节点

G6 的 `render()` 可能触发自动适配，`layout()` 会重新计算已有节点坐标。实时新增子节点时如果继续调用这些流程，用户当前画布位置会被自动改变。

### 初始适配时机过早

刷新页面后，图谱初始化可能早于右侧容器尺寸稳定。此时执行 `fitView` 会以不完整的容器尺寸计算视口，最终出现图谱数据已存在但画面落在空白区域的问题。

### 展开按钮命中区域不稳定

G6 默认端口可见，并且节点 badge 内部元素可能成为点击目标，导致点击事件拿不到稳定的节点 id。视觉上端口也容易和展开加号混淆，造成“第一次点击没展开”的体验。

### 新增节点缺少避让坐标

增量追加子节点时，如果只依赖默认布局或简单固定偏移，新节点容易和同层或跨层已有节点重叠。

## 修复方案

### G6 增量更新节点和边

图谱数据更新改为按差异执行：

- `addNodeData`
- `updateNodeData`
- `removeNodeData`
- `addEdgeData`
- `updateEdgeData`
- `removeEdgeData`

这样 SSE 推送新消息或新事件时，只更新发生变化的节点和边，不再把整张图重新生成一遍，避免 Agent 输出期间图谱闪烁或消失。

### 增量绘制时保持视口不动

新增 `drawWithoutViewportShift()`：

1. 记录当前 `graph.getPosition()` 和 `graph.getZoom()`。
2. 调用 `graph.draw()` 刷新增量数据。
3. 恢复之前的平移位置和缩放比例。

实时新增子节点时不再调用会自动适配视口的 `render()`，也不再调用会重排已有节点的 `layout()`。

### 只允许用户双击画布触发自适应

画布自适应调整只保留两个入口：

- 初次进入页面，在容器尺寸稳定后自动执行一次。
- 用户双击画布时手动执行。

节点双击和实时新增子节点不会触发 `fitView`，避免用户正在查看的区域被自动移动。

### 延迟初始 `fitView`

新增初始适配调度逻辑：

- 等待图谱容器宽高稳定。
- 使用 `initialFitDoneRef` 保证初始自适应只执行一次。
- 在初始化完成但尚未完成初始适配时，延迟执行 `fitView`。

这解决了刷新后右侧图谱初始显示为空白，需要额外点击后才恢复的问题。

### 修复展开按钮命中

展开/折叠交互调整为：

- 隐藏 G6 默认 port：`port: false`。
- 使用右侧独立 badge 表示 `+` / `-`。
- 增强点击命中判断：
  - `elementNodeId()` 向上查找原始 DOM 目标所属节点。
  - `isCollapseToggleEvent()` 同时判断 badge 目标和右侧按钮坐标区域。
  - `eventCanvasPoints()` 兼容不同事件里的 canvas / viewport 坐标。

修复后点击 `ResearchAgent` 右侧加号一次即可展开。

### 新增节点避让排布

新增本地节点摆放与碰撞检测逻辑：

- `nodeRectAt()` 根据节点尺寸计算占位矩形。
- `overlaps()` 判断节点矩形是否相交。
- `nextVerticalOffset()` 生成上下交错的候选偏移。
- `findFreeNodePosition()` 从父节点右侧开始寻找不重叠位置。

同一批新增的多级节点会记住已经分配的位置，后续节点继续基于已占用区域避让，减少子节点与其他节点重叠。

## 影响范围

| 文件 | 改动 |

| `frontend/src/components/agent-flow/AgentFlowGraph.tsx` | 增量更新、视口保持、初始适配、展开按钮命中、节点避让排布 |
| `frontend/src/components/agent-flow/transform.ts` | 图谱节点折叠状态与数据转换配合增量更新 |

## 验证

已完成验证：

```bash
cd frontend
npm run build
```

结果：

- 前端构建通过。
- 刷新研究详情页后，右侧图谱可正常显示，不再初始落在空白区域。
- 点击 `ResearchAgent` 右侧加号一次即可展开，浏览器验证节点数量从 `6/190` 增加到 `9/190`。
- 实时新增子节点时，画布不会自动自适应移动；用户可通过双击画布手动恢复全图视角。
- 前端 `5173` 和后端 `8080` 服务保持正常监听。

## 后续注意事项

- 后续修改 Agent Flow 时，实时更新路径应优先使用 G6 增量 API，避免回到整图 `render()` / `layout()`。
- 如果新增节点类型或改变节点尺寸，需要同步检查避让矩形计算是否仍准确。
- 如果需要自动定位新节点，应设计成显式用户操作或可关闭选项，不应在 SSE 实时追加过程中默认移动用户视口。
- 若后续引入新的 G6 交互插件，需要确认它不会重新启用默认 port 或覆盖当前展开按钮命中逻辑。
