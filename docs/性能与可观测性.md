# 性能与可观测性

## 性能策略

1. Supervisor 根据预算生成有限数量的子研究，并使用 `asyncio.gather` 按 `maxConcurrentUnits` 并发执行。
2. Tavily 查询使用规范化参数作为缓存 key，TTL 缓存避免重复搜索，in-flight 合并避免并发重复请求。
3. 网页摘要按 URL 与内容缓存，并限制摘要并发数、输入字符数和单次超时；失败时降级使用 Tavily 原始摘要。
4. 报告阶段通过 `RESEARCH_REPORT_FINDINGS_MAX_CHARS` 限制汇总材料，避免最终模型调用因上下文膨胀出现长尾延迟。
5. 后台任务使用有界队列和固定 worker，队列满时快速拒绝，避免无界内存增长。

预算默认值：

| 预算 | 子研究 | 每个子研究搜索 | 并发数 | 单次结果 |
|---|---:|---:|---:|---:|
| MEDIUM | 2 | 2 | 1 | 3 |
| HIGH | 4 | 3 | 2 | 3 |
| ULTRA | 6 | 4 | 3 | 3 |

## 可观测链路

OpenTelemetry span 层级：

```text
deep_research.workflow
  -> deep_research.stage <AgentName>
    -> deep_research.model <ModelName>
    -> deep_research.tool <ToolName>
```

公共属性包括 `research.id`、`user.id`、`model.id`、`budget.level`、`workflow.status`、token 用量和调用延迟。Langfuse 使用 OTLP HTTP 导出；也可配置任意兼容接收端。

默认不记录提示词和模型输出。启用 I/O 采集后，系统会按字符上限截断，并对 Authorization、API key、secret 和 token 脱敏。

## 验证

- `pytest` 验证响应、缓存、SSE、观测配置与报告输入上限。
- `sse_smoke.py` 验证真实 HTTP、事件流和结束包。
- `live_workflow_smoke.py` 使用数据库中的 Mimo 配置验证完整状态流转、搜索、报告和持久化。
- Langfuse 中应能按 `research.id` 查到完整 workflow，并展开 stage、model、tool 子 span。
